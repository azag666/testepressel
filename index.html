Guia de Configuração no ManyChat 

            
Siga este passo a passo para configurar corretamente seu ManyChat e integrar com o AZAG System.

            
            

            

                
                

                    
 1. Importe o Fluxo Completo AZAG System no ManyChat

                    
Para facilitar sua integração, criamos um fluxo único que contém todas as requisições e ações necessárias para o AZAG System. Clique no botão abaixo para importar este fluxo diretamente para o seu ManyChat:

                    
                    
                         Importar Fluxo Completo AZAG System
                    

                    
**Visualização do Fluxo Completo:**

                    

                        
                    

                    
Após clicar em "Importar Fluxo Completo AZAG System", o ManyChat será aberto com o fluxo carregado. Este fluxo já contém todas as requisições de API e ações que detalharemos nos próximos passos.

                    
**Importante:** Após a importação, é fundamental clicar no botão Publish (Publicar) no canto superior direito do ManyChat para salvar e ativar o fluxo.

                


                
                

                    
 2. Entendendo as Requisições de API no Fluxo (Já Importadas)

                    
O fluxo que você importou contém três requisições de API principais. Elas são blocos de "Requisição Externa" que se comunicam com o AZAG System. Não é necessário criá-las, apenas entender como funcionam:

                    
                    
                    

                        
A) Ajustar Valor PIX

                        
Esta ação é usada para dividir o campo `valor venda` por 100 e armazenar o resultado em `valor em reais`. Isso é útil para converter valores de centavos para reais, por exemplo.

                        
**Configuração da Ação "Definir campo do usuário" no ManyChat (dentro do fluxo importado):**

                        
                        
Captura de Tela 2025-07-23 às 18.06.04.png

                        
**Ação:** Definir campo do usuário

                        
**Campo:** valor em reais

                        
**Valor:** {{valor venda}}/100

                    


                    
                    

                        
B) Gerar Pagamento (PIX)

                        
Este bloco de "Requisição Externa" é responsável por gerar um código PIX e notificar o AZAG System. Ele é geralmente usado antes de apresentar o PIX ao seu cliente no ManyChat.

                        
**Configuração da Requisição Externa no ManyChat (dentro do fluxo importado):**

                        
                        
Captura de Tela 2025-07-23 às 18.06.50.png

                        
**URL:** https://lead-tracker-api-production.up.railway.app/api/registerPix

                        
**Tipo de Pedido:** POST

                        
**Corpo da Solicitação (JSON):**

                        
{
  "click_id": "{{click_id}}",
  "pix_id": "{{$.id}}",
  "pix_value": "{{valor em reais}}",
  "secretKey": "12341"
}

                        
**Atenção ao `click_id`:** Este campo é crucial para o rastreamento. Ele já deve estar configurado no fluxo. Se você usa anúncios do Facebook, a variável `{{ad_id}}` é ideal. Caso contrário, `{{flow_json.click_id}}` funciona perfeitamente.

                        
Mapeie a resposta da API para os campos personalizados: `$.id` (para o ID do PIX) e `$.qr_code` (para o QR Code do PIX). Estes mapeamentos já vêm prontos no fluxo importado.

                    


                    
                    

                        
C) Confirmar Pagamento

                        
Este bloco de "Requisição Externa" notifica o AZAG System de que um pagamento foi confirmado. Ele é ideal para ser usado após a identificação de um PIX pago (por exemplo, quando sua plataforma de pagamento envia um webhook).

                        
**Configuração da Requisição Externa no ManyChat (dentro do fluxo importado):**

                        
                        
Captura de Tela 2025-07-23 às 18.07.43.png

                        
**URL:** https://lead-tracker-api-production.up.railway.app/api/confirmPayment

                        
**Tipo de Pedido:** POST

                        
**Corpo da Solicitação (JSON):**

                        
{
  "click_id": "{{click_id}}",
  "conversion_value": "{{valor em reais}}",
  "secretKey": "12341"
}

                        
Certifique-se de que click_id e valor em reais são os campos personalizados do ManyChat. Eles já devem estar configurados corretamente no fluxo importado.

                    

                


                
                

                    
 3. Verifique e Crie os Campos Personalizados (Se Necessário)

                    
Campos Personalizados são como "gavetas" no ManyChat onde guardamos informações importantes sobre seus clientes (como o valor do PIX, o status do pagamento, etc.). **Quando você importa o fluxo completo, o ManyChat geralmente cria essas "gavetas" automaticamente para você.** Mas é sempre bom dar uma olhada para ter certeza de que está tudo certo!

                    
Para verificar, siga estes passos simples no seu ManyChat:

                    
    
                        
  1. No menu lateral esquerdo do ManyChat, clique em Settings (Configurações).
    2. 
                        
  2. Em seguida, clique em Custom Fields (Campos Personalizados).
    3. 
                        
  3. Você deve ver uma lista de campos. Verifique se os seguintes campos de usuário existem e se o "Tipo" (Type) está correto:
    4. 
                    

                    
    
                        
  • **valor em reais**: Tipo **Number**
    • 
                        
  • **valor da venda**: Tipo **Number**
    • 
                        
  • **$.id**: Tipo **Text**
    • 
                        
  • **$.qr_code**: Tipo **Text**
    • 
                        
  • **$.status**: Tipo **Text**
    • 
                    

                    
                    
Captura de Tela 2025-07-23 às 19.03.53.png

                    
**Se algum campo estiver faltando, você pode criá-lo manualmente:**

                    
    
                        
  1. Na mesma tela de Custom Fields, clique no botão azul + New User Field.
    2. 
                        
  2. No campo 'Field Name' (Nome do Campo), digite o nome exato do campo que está faltando (ex: `valor em reais` ou `$.id`).
    3. 
                        
  3. Em 'Field Type' (Tipo do Campo), escolha o tipo correto (use **Number** para valores numéricos e **Text** para IDs e QR Codes).
    4. 
                        
  4. Clique em 'Create' (Criar).
    5. 
                    

                    
**Importante:** Preste muita atenção aos nomes dos campos, incluindo o `$.` para os campos `$.id`, `$.qr_code` e `$.status`. O ManyChat é sensível a isso, mas geralmente aceita o `$.` sem problemas.
